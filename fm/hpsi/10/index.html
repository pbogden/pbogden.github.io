Mobile demo
The smallest browsers (e.g., iPhone 5) have a screen width of 320px. Since the FM website puts 45px borders on each side (30px top/bottom), the content must fit in a 230px-wide container. In this demo, both the container and graphic are responsive so the text hugs the bottom of the graphic.
Mobile fixed-height demo
This demo shows what happens with the current version of the FM site, which is not fully responsive. Instead, the site imposes a fixed-height constraint on the graphic's container. For desktops, this isn't noticeable. But on narrow devices (such as iPhones) the text no longer hugs the bottom of the graphic. Instead, there's a big blank space that ends up being about the same size as the graphic.
Responsive demo
This shows what happens when the container for the graphic is fully responsive. On wide screens, the text goes on the right. On narrow screens, it moves to the bottom of the graphic with no blank space. Regardless of browser size, the graphic fills the container. You can adjust the width of your browser to see the effect. Eventually, the FM site will behave this way.

How it works: This demo uses a div container that contains both the graphic and a PNG. The PNG determines the container dimensions, which makes the container fully responsive. The FM site behaves this way for for images, but not yet for the interactive graphics. The solution demonstrated here is to put the graphic in the same container as (and on top of) the PNG. This requires only one line of HTML code. When the main page loads or resizes, a few lines of JavaScript detect the new container dimensions (as determined by the PNG), and reset the height and width of the graphic accordingly.
